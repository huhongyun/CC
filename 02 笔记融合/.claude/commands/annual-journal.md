# 生成年记

从已有的月记中综合提炼，生成年度日记并保存到印象笔记。

## 参数

- `$ARGUMENTS` — 可选，指定年份，格式 `YYYY`。不指定则自动取去年。

## 执行步骤

### 1. 确定年份

如果用户提供了 `$ARGUMENTS`，解析为年份。

如果没有提供，通过 resolve 命令自动计算：
```bash
python3.11 weekly_journal.py resolve --type year
```
输出去年年份。

如果用户指定了年份：
```bash
python3.11 weekly_journal.py resolve --type year --date YYYY
```

### 2. 读取月记

优先从本地 `output/journal_YYYYMM.md` 读取。用 Python glob 精确过滤该年份的月份文件：

```bash
python3.11 -c "import glob; files=sorted(glob.glob('output/journal_YYYY*.md')); [print(f) for f in files]"
```

如果本地文件不完整（少于12份），从印象笔记拉取对应月份的晨间日记并先生成缺失的月记。兼容不完整年份（按现有月记数量生成年记，不完全依赖12份）。

### 3. 生成年记

读取月记内容，按以下格式生成年记 Markdown：

```markdown
# 年记 | YYYY年

## 一、年度概览

（用 3-4 句话概括全年最重要的转变和成长）

---

## 二、季度回顾

### 第一季度（1~3月）| 主题
（综合 1-3 月月记，提炼季度主线，5-8 句话）

### 第二季度（4~6月）| 主题
...

### 第三季度（7~9月）| 主题
...

### 第四季度（10~12月）| 主题
...

---

## 三、年度关键主题

### 1. 主题名称
（从月记中提炼 5 个贯穿全年的核心主题，每个 5-8 句话，覆盖全年演变轨迹）

### 2. 主题名称
...

---

## 四、成长轨迹

### 认知升级
（全年认知/学习/哲学层面的突破和演变）

### 身体管理
（全年运动、健康、养生的变化轨迹）

### 工作推进
（全年项目进展、团队成长、技术突破）

### 家庭关系
（全年家庭事件、关系变化、情感脉络）

---

## 五、年度数据

| 指标 | 数据 |
|------|------|
| （从月记中提取可量化的年度数据） |

---

## 六、下一年展望

1. **展望1**
2. **展望2**
3. **展望3**
4. **展望4**
5. **展望5**
```

**写作要求**：
- 基于月记原文事实，不添加虚构内容
- 季度回顾应有连贯叙事，不是月记的简单拼接
- 年度关键主题应展示全年演变轨迹（Q1→Q2→Q3→Q4），而非孤立描述
- 成长轨迹应有对比（年初 vs 年末），体现变化
- 年度数据尽量量化（项目数、里程数、笔记数等）
- 下一年展望从月记"下月关注"中反复出现的未完结课题中提取

### 4. 保存到印象笔记

将生成的 Markdown 写入 `output/journal_YYYY.md`，然后运行：

```bash
PYTHONIOENCODING=utf-8 python3.11 weekly_journal.py save --title "YYYY~年记" --file output/journal_YYYY.md
```

标签会根据标题自动添加（含 `~年记` 自动加 `年记` 标签）。

### 5. 输出结果

告知用户：
- 综合了多少份月记
- 年记保存的位置和印象笔记 GUID
- 全年的 3 个核心主题（一句话）
